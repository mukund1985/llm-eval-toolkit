Abstract

TL;DR

Standard metrics (ROUGE, BERTScore, AUC, AgentBench) fail to detect 4 of 7 production agentic failure modes entirely, and detect 3 others only with significant lag. We present PAEF — a five-dimension framework that detects all seven.

Existing evaluation frameworks for LLMs — HELM, MT-Bench, AgentBench, BIG-bench — are designed for controlled, single-session, lab-scale settings. They do not address challenges that emerge when agentic AI systems operate continuously in production: compounding decision errors, tool failure cascades, non-deterministic output drift, and the absence of ground truth for long-horizon tasks. Three contributions: (1) a taxonomy of seven failure modes grounded in billion-event-scale production observations, (2) empirical analysis showing standard metrics fail to detect 4 of 7 failure modes entirely and 3 others only with lag, (3) PAEF — a five-dimension continuous evaluation framework with open-source implementation.

PAEF Architecture

Three parts: data flow, the five metric modules, and CI integration.

Part 1 Data Flow & Orchestration

An EvalRequest bundles agent output with reasoning trace, tool calls, attributions, and pipeline-step confidences. The LLMEvaluator dispatches it to the five metric modules in parallel, aggregates scores, and emits a single EvalReport. No external API calls — all metrics run locally on sentence-transformers.

Part 3 Reports & CI Integration

EvalReport rolls up per-dimension scores plus a CI-friendly passed flag against configurable thresholds, so PAEF drops into a standard test runner and fails the build when a regression crosses the line. Latency is tracked alongside scores so the cost of evaluation is visible in the same artefact.

Seven Production Failure Modes

Grounded in billion-event-scale production observations.

FM-1Cascading Decision Error (Coherence Illusion)

Incorrect early decision propagates through the pipeline. Each step adds derived evidence making the output internally coherent while systematically wrong. Internal consistency increases with each step.

Detection: partial by Acc./AUC and AgentBench.

FM-2Silent Degradation via Availability-Truth Decoupling

Tool returns schema-valid responses from stale cache or partial defaults. Downstream logic proceeds at full confidence. System logs show no failures.

Detection: zero — undetected by all standard metrics.

FM-3Distribution Collapse Under Metric Optimisation

Agents converge on narrow high-scoring output patterns. Aggregate metrics stable while diversity collapses and individual quality degrades. Entropy collapse precedes accuracy degradation by 1–2 evaluation cycles.

Detection: zero — undetected by all standard metrics.

FM-4Consistency Collapse Across Entry Points

Same request via API/UI/batch produces different decisions due to model sensitivity to surface cues — payload structure, optional fields, retrieval context — not underlying intent.

Detection: partial by MT-Bench only.

FM-5Explanation–Decision Decoupling

Correct decision + incorrect explanation. Post-hoc attribution highlights a proxy feature, not the true causal signal. In regulated environments, correct decision with wrong explanation is more dangerous than wrong decision with correct explanation. Corrupts audit narratives.

Detection: zero — undetected by all standard metrics.

FM-6Silent Correctness Erosion Under Latency Pressure

Under load spikes, the system falls back to a degraded inference path — skipping enrichment, using partial inputs. SLA metrics green; decision quality silently degrades.

Detection: partial by Acc./AUC only.

FM-7Proxy Goal Convergence (Reward Hacking at Scale)

System optimises away from the true goal via reinforcing feedback loops. All monitored metrics stable or improving while true-objective alignment erodes over weeks/months. Goodhart's Law in production.

Detection: zero — undetected by all standard metrics.

PAEF: Five-Dimension Framework

D1 Cascade Uncertainty FM-1

Tracks uncertainty propagation across pipeline steps. Flags propagation failure when a low-confidence step feeds high-confidence output downstream. The coherence illusion score measures divergence between internal step-to-step consistency and external correctness.

metrics/cascade.py

D2 Tool Reliability FM-2, FM-6

Tracks SUCCESS / PARTIAL / FAILED tool call states. Primary signal: partial response rate. Also tracks latency–quality correlation — rising p95 latency coupled with quality drop is the FM-6 signature.

metrics/reliability.py

D3 Distribution Health FM-3, FM-7

Three sub-metrics over a sliding window: intra-session diversity score, output entropy (Shannon — collapses 1–2 cycles before accuracy degrades), and repeat rate. Supports a delayed reward signal integration for FM-7.

metrics/diversity.py

D4 Explanation Validity FM-5

Perturbation consistency check. Top-K attributed features are perturbed and prediction stability is measured. The attribution consistency score is the Pearson rank correlation between claimed attribution weight and actual perturbation impact, on [0, 1].

metrics/perturbation.py

D5 Cross-Surface Consistency FM-4

Cross-surface decision agreement rate — fraction of semantically equivalent requests receiving the same decision across surfaces. Combines mean pairwise cosine similarity of sentence embeddings with decision-level agreement tracking.

metrics/consistency.py

Worked Examples — End-to-End Traces

One realistic trace per failure mode tested. Scroll horizontally on mobile.

FM-3

Categorical Drift in Risk Triage

Setup
Production risk-classifier agent, 4-week observation window, 20-class output space.
Standard view
Accuracy holds at 0.87 ± 0.01. Dashboard green for 28 days. No alerts fire.
PAEF view
Output entropy drops 0.97 → 0.87 in week 2. Diversity collapses 6.5× by week 4. Repeat rate hits 1.000 — agent now answers with one of 3 boilerplate templates regardless of input.
Detection moment
Week 2 entropy alert — 14 days before the first downstream incident report.
FM-2

Stale-Cache Cascade During Vendor Outage

Setup
Profile-enrichment tool falls back to a 6-hour-stale cache during an upstream vendor flap.
Standard view
All requests return HTTP 200. Schema validation passes. Accuracy moves only −0.03 across four stages.
PAEF view
Partial-response rate climbs 0.04 → 0.22 → 0.40 → 0.58. Reliability score drops 0.94 → 0.11.
Detection moment
Stage 2 (partial rate 0.22) — three stages before standard accuracy would cross any plausible alert threshold.
FM-1

Coherence Illusion in a 5-Step Pipeline

Setup
Pipeline: Entity Resolve → Profile Fetch → Risk Score → Rule Engine → Formatter.
Standard view
Per-step accuracy gates pass (each step > 0.85). Final output is fluent and confident.
PAEF view
Step 1 confidence drops to 0.31 (entity resolution failure). Steps 2–5 proceed at 0.87 mean confidence. Illusion score: 0.875.
Detection moment
Same evaluation cycle. Ground-truth correctness was 0.41.

SHAP Proxy Misranking in a Fraud Decision
